02 · Mathematisches Fundament

Pyramide mit n Ecken: Aufbau

Die Pyramide wird vollständig durch vier Parameter beschrieben. Alle Eckpunkte entstehen als Vektoren.

Parameter

n  — Anzahl der Grundflächen-Ecken (n ≥ 3)

r  — Umkreisradius des Grundpolygons

h  — Höhe der Pyramide (Spitze → Grundebene)

PApex — Spitze, verschiebbar in x/y/z

Eckpunkte als Vektoren

Für i = 1 … n werden n gleichmäßige Eckpunkte auf dem Umkreis erzeugt. Spitze: PApex = {x, y, h}. Daraus entstehen n Seitendreiecke:

D[i] = {EckpunkteGF[[i]],
       EckpunkteGF[[i+1]], PApex}

Kreuzprodukt, Fläche & Volumen

Flächenberechnung via Kreuzprodukt

Für jedes Seitendreieck werden zwei Kantenvektoren gebildet. Das Kreuzprodukt liefert direkt die Parallelogrammfläche, geteilt durch 2 ergibt die Dreiecksfläche:

// Kantenvektoren des i-ten Seitendreiecks a = EckpunkteGF[[i+1]] - EckpunkteGF[[i]] b = PApex - EckpunkteGF[[i]] // Fläche des Seitendreiecks A_i = (1/2) * ‖a × b‖

Warum liefert das Kreuzprodukt die Fläche?

‖a × b‖ = |a| · |b| · sin(θ) → Basis · Höhe des Parallelogramms Parallelogramm ÷ 2 → Seitendreieck × n → Mantelfläche A_M = ∑ ½ · ‖aᵢ × bᵢ‖, i = 1 … n

Das Kreuzprodukt liefert die Fläche, ohne die Dreieckshöhe separat berechnen zu müssen.

Volumenformel

A_Grund = (n · r² / 2) · sin(2π/n) // Fläche des n-Ecks V = (1/3) · A_Grund · h // Pyramidenvolumen → V(n, r, h)

Grenzwertidee: n → ∞

Was passiert, wenn die Pyramide immer mehr Ecken bekommt?

Kernaussage: Der Grenzwert n → ∞ ist kein Sprung ins Kontinuierliche, sondern ein messbarer, kontrollierbarer Prozess. Genau das macht das Notebook sichtbar.

Vektoren im Code: ErzeugePyramide

(* Grundpolygon: n Punkte auf Umkreis mit Radius r *) EckpunkteGF = CirclePoints[Radius, AnzahlEckPunkte]; EckpunkteGF = Append[#, 0] & /@ EckpunkteGF; (* Spitze parametrisch – PApex = {sx, sy, Hoehe} *) (* Vektoriell: Seitendreiecke aufbauen *) ListeSeitenDreiecke = Table[ {EckpunkteGF[[i]], EckpunkteGF[[i+1]], PApex}, {i, 1, AnzahlEckPunkte-1} ]; AppendTo[ListeSeitenDreiecke, {Last[EckpunkteGF], First[EckpunkteGF], PApex}];
Warum CirclePoints? CirclePoints gibt gleichmäßig verteilte Punkte auf dem Einheitskreis zurück, mathematisch das diskrete Analogon zur Parameterdarstellung des Kreises. Das macht die Brücke zum Grenzwert explizit.

Ein Prinzip – viele Anwendungen

Der Grenzwert n → ∞ ist kein Sprung, sondern ein messbarer, kontrollierbarer Prozess. Dasselbe Prinzip taucht an drei Stellen der Mathematikgeschichte auf.

Archimedes & die Exhaustionsmethode

Archimedes (287–212 v. Chr.) näherte Kreisflächen durch einbeschriebene Polygone an, exakt die Idee hinter dem Notebook. Er berechnete damit als erster eine verlässliche Schranke für π.

Die Riemannsche Summe

Die Idee

Intervall [a, b] wird in n gleich breite Streifen geteilt: Δx = (b−a)/n. Rechteck der Höhe f(xᵢ) auf jedem Streifen:

Sₙ = ∑ f(xᵢ) · Δx

Für n→∞ wird Δx infinitesimal: lim Sₙ = ∫ f(x) dx

Verbindung zur Pyramide

Pyramide: n Dreiecke → Kegel.
Riemannsche Summe: n Rechtecke → Integral.

Dasselbe Prinzip – Approximation durch Zerlegung – in zwei Dimensionen. Bernhard Riemann (1826–1866) formalisierte den Integralbegriff durch genau dieses Grenzwertprinzip.

06 · Reflexion

Was hat diese Methode gebracht?

Drei Fragen, ehrlich beantwortet.

Was bringt Visualisierung gegenüber reiner Formelherleitung?

Sie macht den Grenzprozess erfahrbar. Man beobachtet Konvergenz, statt sie nur zu berechnen. Das verankert Intuition und erleichtert das Lernen, was algebraische Umformungen allein nicht leisten können.

Wo stößt Visualisierung an Grenzen?

Sie ersetzt keinen Beweis. n = 100 sieht aus wie ein Kegel, ist es aber streng genommen nicht. Die formale Grenzwertaussage braucht Analysis, nicht nur Bilder.

Was würde eine reine Formelherleitung nicht liefern?

  • →Die Visualisierung der Konvergenz – man sieht, wie das Polygon zur Kurve wird.
  • →Die Abhängigkeit von r und h – wie verändern sich Fläche und Volumen mit den Parametern?
  • →Die asymmetrische Wirkung der Spitzenverschiebung — Mantelfläche wächst, Volumen bleibt konstant: erst durch Interaktion greifbar.
Fazit der Reflexion: Visualisierung nimmt eine Mittelstellung ein. Sie ersetzt den formalen Beweis nicht, ist aber mehr als bloße Illustration. Sie macht Konvergenz beobachtbar statt nur berechenbar. Diese epistemische Rolle ist selbst mathematisch bedeutsam.

07 · Abschluss

Antworten auf die Einstiegsfragen

Die vektorielle Beschreibung erlaubt es, Pyramiden beliebiger Ordnung einheitlich zu parametrisieren.

F1: Wie kann man mit Geraden Grenzformen darstellen?

Der Grenzwert n → ∞ ist kein Sprung ins Kontinuierliche, sondern ein messbarer Prozess: Für n = 26 beträgt der Fehler bereits weniger als 1 %. Vektoriell beschriebene Seitendreiecke approximieren den Kegelmantel, die Grundfläche nähert sich der Kreisfläche. Geradlinige Objekte bilden krummlinige Grenzformen.

F2: Kann eine Visualisierung als mathematischer Beweis gelten?

Umstritten. Formal ersetzt ein visuelles Argument keinen analytischen Beweis: n = 100 sieht aus wie ein Kegel, ist es streng genommen aber nicht. Seit dem Aufkommen computergestützter Mathematik wird die Grenze zwischen Evidenz und Beweis jedoch aktiv diskutiert. Visualisierung nimmt eine Mittelstellung ein: Sie ersetzt den formalen Beweis nicht, ist aber mehr als bloße Illustration. Diese epistemische Rolle ist selbst mathematisch bedeutsam.

Drei Werkzeuge. Ein Gedanke.
